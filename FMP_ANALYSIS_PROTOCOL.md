# FMP Analysis Protocol

## Purpose

This file defines how Fractal Metascience Paradigm analysis should be applied
to directives, protocols, repository surfaces, and public documentation inside
the Terra ecosystem.

It does not replace validation, audit, or legal review.
It adds a distinct structural-analysis layer.

## Why this file exists

Without a separate FMP analysis layer, two distortions happen:

- validation is reduced to generic compliance language;
- structural analysis is reduced to loose commentary.

This protocol keeps FMP analysis readable, reusable, and donor-grade.

## Scope

Use this protocol when analyzing:

- directives;
- sequence and workflow protocols;
- repository governance surfaces;
- publication-facing bridge files;
- continuity and recovery logic;
- points where hidden fragmentation produces public disorder.

## Core rule

FMP analysis must not act like opinion-first commentary.
It must identify structure, signal breaks in continuity, and feed back into
repair.

## Minimal analysis sequence

1. Define the analyzed surface.
2. Identify its declared role.
3. Identify its actual behavior in the ecosystem.
4. Mark contradictions, fractures, duplications, or false claims of canonicity.
5. Distinguish:
   - canonical layer;
   - donor layer;
   - operational layer;
   - archive or shadow layer.
6. State what kind of repair is needed:
   - detox;
   - clarification;
   - extraction;
   - relocation;
   - canonicalization;
   - deprecation.

## Required output

An FMP analysis output should name:

- the target surface;
- the structural role of that surface;
- the detected fragmentation or distortion;
- the likely origin of the distortion;
- the recommended correction path.

## Prohibited shortcuts

Do not:

- treat machine-facing files as if they were human entry documents;
- declare a file canonical only because it exists at repository root;
- confuse mirrored residue with live donor material;
- replace reading with auto-generated summary language;
- present unresolved structural tension as if it were already solved.

## Human-first rule

FMP analysis should increase clarity for human readers and maintainers.
It must not multiply maps, labels, or abstractions unless they reduce real
fragmentation.

## Working conclusion

The purpose of FMP analysis is not to decorate the ecosystem with meta-language.
Its purpose is to detect fragmentation, preserve continuity, and return the
system to a truer structure.
